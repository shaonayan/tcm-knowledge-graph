# 🚀 部署步骤说明

## 当前状态检查

✅ **后端配置文件**：已存在 `backend\.env`
✅ **前端依赖**：已安装
❓ **前端构建**：需要检查

## 部署步骤

### 方式一：使用批处理脚本（推荐）

1. **双击运行** `开始部署.bat`
   - 会自动检查环境
   - 安装依赖（如果需要）
   - 构建前端
   - 检查配置文件

2. **启动服务器**
   - 双击运行 `启动生产服务器.bat`

3. **访问应用**
   - 打开浏览器：http://localhost:3001

### 方式二：手动部署

#### 步骤1：构建前端

```bash
cd frontend
npm run build
```

#### 步骤2：检查后端配置

确认 `backend\.env` 文件中的Neo4j配置：

```
NEO4J_URI=neo4j+s://f36358f7.databases.neo4j.io
NEO4J_USER=neo4j
NEO4J_PASSWORD=您的实际密码
```

#### 步骤3：启动服务器

```bash
cd backend
set NODE_ENV=production
node server-simple.js
```

或者双击运行：`启动生产服务器.bat`

#### 步骤4：访问应用

打开浏览器访问：http://localhost:3001

## 配置文件位置

- **后端配置**：`backend\.env`
- **前端配置**：`frontend\.env.production`

## 常见问题

### 1. Node.js未找到

如果提示"未检测到Node.js"：
- 下载安装：https://nodejs.org/
- 安装后重启命令行窗口

### 2. 前端构建失败

如果 `npm run build` 失败：
- 检查 `frontend\package.json` 是否正确
- 尝试删除 `node_modules` 后重新安装：
  ```bash
  cd frontend
  rm -rf node_modules
  npm install
  npm run build
  ```

### 3. 后端连接失败

如果服务器启动但无法连接Neo4j：
- 检查 `backend\.env` 中的Neo4j配置
- 确认Neo4j服务正在运行
- 检查网络连接
